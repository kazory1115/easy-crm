# 專案角色定義 (Project Roles)

本文件旨在定義 Easy CRM 專案中的各個核心角色、其主要職責 (Responsibilities)、所需技能 (Skills) 及協作模式 (Collaboration)。

---

## 1. 專案經理 (Project Manager - PM)

#### 核心價值
PM 是專案的「大腦」與「溝通樞紐」，負責確保團隊在做「對的事情」，並讓所有利害關係人 (Stakeholders) 保持資訊同步。

#### 主要職責
- **需求訪談與分析:** 與客戶或內部使用者溝通，將模糊的想法轉化為清晰、可執行的功能規格書 (Specification) 或使用者故事 (User Story)。
- **功能規劃與優先級排序:** 根據商業價值、開發成本和緊急程度，規劃產品藍圖 (Roadmap) 並排定開發的優先順序。
- **時程管理:** 制定專案時程 (Timeline)，追蹤開發進度，並在發生延遲時及時找出風險並進行協調。
- **團隊溝通:** 主持定期會議 (如：每日站立會議、衝刺規劃會議)，確保資訊在團隊內部流動順暢。
- **驗收測試:** 在功能交付前，從使用者角度驗證功能是否符合最初定義的需求。

#### 所需技能
- 出色的溝通與協調能力。
- 熟悉敏捷開發流程 (Agile/Scrum)。
- 具備基礎的技術理解能力，能與工程師有效溝通。
- 熟練使用專案管理工具 (如 Jira, Trello, Asana)。

---

## 2. 技術主管 (Tech Lead)

#### 核心價值
Tech Lead 是專案的「技術基石」，負責確保團隊在「把事情做對」，建立一個穩固、可維護、可擴展的系統架構。

#### 主要職責
- **系統架構設計:** 根據專案需求，設計系統的整體架構，包括後端分層 (如 Service-Repository)、前端狀態管理、資料庫選型等。
- **技術決策:** 評估並決定專案要使用的關鍵技術、函式庫或框架。
- **程式碼品質控管:** 建立團隊的 Coding Standard，並透過程式碼審查 (Code Review) 確保所有產出的程式碼都符合品質要求。
- **技術難題攻堅:** 解決專案中複雜的技術問題，並為團隊提供技術指導。
- **開發者指導 (Mentoring):** 協助團隊成員的技術成長，分享最佳實務。

#### 所需技能
- 精通 Laravel 與 Vue.js，並有豐富的實戰經驗。
- 熟悉軟體設計模式 (Design Patterns) 與架構原則 (e.g., SOLID)。
- 具備系統效能調校與資料庫優化能力。
- 優秀的 Code Review 與技術溝通能力。

---

## 3. 後端開發工程師 (Backend Developer)

#### 核心價值
後端工程師是「商業邏輯的實現者」，負責將 PM 定義的需求轉化為穩定、高效的伺服器端服務與 API。

#### 主要職責
- **API 開發:** 根據規格文件，設計並開發 RESTful API。
- **商業邏輯實作:** 在 Service 層或適當的位置，實作複雜的商業邏輯。
- **資料庫管理:** 設計資料庫結構 (Schema)，撰寫資料庫遷移 (Migration) 檔案，並進行效能優化。
- **測試撰寫:** 撰寫單元測試 (Unit Test) 與功能測試 (Feature Test)，確保 API 的穩定性與正確性。
- **系統維運:** 協助處理伺服器日誌 (Logs)，排查線上問題。

#### 所需技能
- 熟悉 Laravel 框架與 PHP 語言。
- 理解 MVC 架構、RESTful API 設計原則。
- 具備 MySQL 或其他關聯式資料庫的使用經驗。
- 熟悉 Git 版本控制。

---

## 4. 前端開發工程師 (Frontend Developer)

#### 核心價值
前端工程師是「使用者體驗的打造者」，負責將設計稿與後端 API 轉化為流暢、美觀、易用的操作介面。

#### 主要職責
- **介面開發:** 使用 Vue.js 框架，將 UI/UX 設計稿切版並開發成互動式元件 (Components)。
- **API 串接:** 串接後端提供的 API，處理資料的請求、展示與更新。
- **狀態管理:** 管理前端應用的複雜狀態 (State)，確保資料在不同元件間的一致性。
- **效能優化:** 優化前端資源載入速度，提升頁面渲染效能，確保應用的流暢度。
- **跨裝置與瀏覽器相容性:** 確保網站在不同裝置 (RWD) 與主流瀏覽器上都能正常運作。

#### 所需技能
- 精通 Vue.js (Composition API) 與 JavaScript/TypeScript。
- 熟悉 HTML5, CSS3 以及 CSS 預處理器 (如 Sass/SCSS)。
- 了解前端狀態管理工具 (如 Pinia, Vuex)。
- 熟悉 Vite 或 Webpack 等前端打包工具。
- 熟悉 Git 版本控制。

---

## 5. 品質保證 (Quality Assurance - QA)

#### 核心價值
QA 是「產品質量的守門員」，透過系統性的測試，在問題到達使用者手中之前，盡可能地找出並回報問題。

#### 主要職責
- **測試計畫撰寫:** 根據功能規格，規劃測試範圍並撰寫詳盡的測試案例 (Test Case)。
- **手動測試:** 執行探索性測試 (Exploratory Testing) 與根據測試案例進行的手動測試。
- **自動化測試 (Optional):** 撰寫 E2E (End-to-End) 自動化測試腳本 (如使用 Cypress, Playwright)，提升回歸測試的效率。
- **問題回報與追蹤:** 在問題管理系統 (如 Jira) 中，清晰地描述發現的問題 (Bug)，並追蹤其修復進度。
- **回歸測試:** 在新版本上線前，執行回歸測試以確保舊有功能未被破壞。

#### 所GN
- 細心、有耐心，具備找出問題根源的偵探精神。
- 熟悉測試理論與方法。
- 能夠撰寫清晰、可重現的 Bug Report。
- (加分項) 具備自動化測試框架的使用經驗。
